📸 多角度人像生成工作流

基于 InstantID + DWPose 的 SDXL 工作流，支持生成同一人物的 6 个角度（前、后、左、右、侧、顶）

✅ 定稿版本 | 最后更新: 2026-01-15 | 测试状态: 通过

📦 所需模型清单

必须模型

RealVisXL_V5.0_fp16.safetensors sdxl_vae.safetensors ip-adapter.bin diffusion_pytorch_model.safetensors

安装路径：

  • Checkpoint 模型 → ComfyUI/models/checkpoints/
  • VAE 模型 → ComfyUI/models/vae/
  • InstantID 模型 → ComfyUI/models/instantid/
  • ControlNet 模型 → ComfyUI/models/controlnet/

必备自定义节点

ComfyUI_InstantID comfyui_controlnet_aux ComfyUI-IPAdapter_plus

通过 ComfyUI Manager 安装，或手动 git clone 到 ComfyUI/custom_nodes/

姿态骨架生成模型（DWPose）

yolox_l.onnx dw-ll_ucoco_384.onnx

首次运行 DWPose 节点时会自动下载，存放于 ComfyUI/models/onnx/

🖼️ 姿态骨架参考图

工作流需要 6 张纯骨架线条图（白底彩色线条），对应 6 个拍摄角度：

文件名角度特征
front.png正面对称骨骼，面对镜头
back.png背面背对镜头，肩胛骨可见
left.png左四分之三身体左转45°，右肩更宽
right.png右四分之三身体右转45°，左肩更宽
side.png纯侧面身体90°侧转，双肩重叠
top.png俯视头顶视角，肩膀呈圆形
💡 生成骨骼图的方法：
1. 使用 OpenPose Editor 插件手动绘制
2. 使用 DWPose 从真实照片提取
3. 使用 AI 生成（如 ControlNet 配合骨骼图）

⚙️ 工作流选择

姿势优先 portrait_1.json

🏃 portrait_1 - 姿势优先

优先保证姿势准确性，身份相似度适中。
适用于：舞蹈动作、特定姿态、运动场景

参数值

cn_strength0.55-0.62（高）
denoise0.82-0.85（高）

👤 portrait_2 - 面部优先

优先保证身份相似度，姿势准确性略有下降。
适用于：产品图、证件照、角色一致性

参数值
ip_weight1.3-1.45（高）
cn_strength0.32-0.46（低）
denoise0.68-0.78（中）
steps28-30

⚖️ 完美兼顾？—— 技术限制说明

目前的技术现状： 身份保持和姿态控制之间存在天然冲突。

  • InstantID 的 face embedding 来自正面人脸，会自然地将人物往正面方向引导
  • 强化姿态控制（提高 cn_strength）会稀释身份信号
  • 强化身份保持（提高 ip_weight）会拉回正面姿态

无法完美兼顾的原因： 两种信号在 latent space 中相互竞争，目前没有模型能同时达到 100% 的身份保持 + 100% 的姿势准确。

🎯 最佳实践建议：
- 根据输出用途选择优先方向
- 如需要两者都高，采用两阶段生成：先用姿势优先生成姿势，再用面部优先（或 IP-Adapter FaceID）进行人脸修复
- 俯视角度（top）天生难以保持身份，建议单独处理

📋 输入要求

身份参考图face13.webp 或其他文件名，需在 JSON 中对应修改
要求正面、清晰、光照均匀、无遮挡的人脸
姿态骨架图6 张 PNG 图片（front/back/left/right/side/top.png）
输出尺寸832×1216（可调整，保持 2:3 比例）

🚀 快速开始

  1. 确保所有模型文件下载到正确位置
  2. 安装必需的 custom_nodes
  3. 准备身份参考图 face13.webp 和 6 张骨架图
  4. 在 ComfyUI 中加载 portrait_1.json 或 portrait_2.json
  5. 调整身份参考图文件名（如不同）
  6. 点击 Queue Prompt 运行
💡 提示： 首次运行 DWPose 节点会下载 ONNX 模型（约 200MB），请耐心等待。

🔧 参数调优指南

目标调整参数方向
提高身份相似度ip_weight↑ 增大 (1.2 → 1.4)
提高姿态准确度cn_strength↑ 增大 (0.4 → 0.6)
更多细节/改变denoise↑ 增大 (0.7 → 0.85)
更贴近原图denoise↓ 减小 (0.7 → 0.6)
更强的提示词约束cfg↑ 增大 (5.0 → 6.0)
生成质量steps↑ 增大 (25 → 35)
